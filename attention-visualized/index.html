01

Why the old approach wasn't enough

Pourquoi l'ancienne approche ne suffisait pas

For a language model to understand a sentence, it needs to know what each word means in context. The word "bank" means something different depending on what surrounds it: "river bank" versus "bank account". So the model needs a way to connect any word to any other word in the sentence, regardless of distance.

Pour qu'un modèle de langage comprenne une phrase, il doit savoir ce que chaque mot signifie en contexte. Le mot "avocat" signifie autre chose selon son entourage. Le modèle a donc besoin d'un moyen de connecter n'importe quel mot à n'importe quel autre dans la phrase, quelle que soit la distance.

Before transformers, models processed text sequentially, one token at a time from left to right. Recurrent networks (RNNs, LSTMs) maintained a hidden state that carried context forward as they moved through the sequence. The problem was that by the time the model reached token 50, the signal from token 1 had been diluted through 49 multiplicative steps. Long-range relationships were very hard to preserve.

Avant les transformers, les modèles traitaient le texte de façon séquentielle, un token à la fois de gauche à droite. Les réseaux récurrents (RNN, LSTM) maintenaient un état caché qui transportait le contexte au fil de la séquence. Le problème : au token 50, le signal du token 1 avait été dilué à travers 49 étapes multiplicatives. Les dépendances longue portée étaient très difficiles à préserver.

⏩
Sequential processing
Traitement séquentiel
RNNs read one token at a time. Processing cannot be parallelized across a sequence, which makes training on long texts extremely slow.
Les RNN lisent un token à la fois. Le traitement ne peut pas être parallélisé sur une séquence, ce qui rend l'entraînement sur des textes longs extrêmement lent.
📉
Vanishing gradients
Gradients qui s'évanouissent
Gradients shrink exponentially as they travel back through long sequences. Early tokens end up receiving almost no useful learning signal.
Les gradients diminuent exponentiellement lorsqu'ils remontent à travers de longues séquences. Les premiers tokens finissent par ne recevoir presque aucun signal d'apprentissage utile.
🔗
The attention fix
La solution attention
Attention lets every token directly connect to every other token in a single parallel step. No bottleneck, no sequential dependency, and the full sequence is processed at once.
L'attention permet à chaque token de se connecter directement à chaque autre en une seule étape parallèle. Pas de goulot d'étranglement, pas de dépendance séquentielle, et toute la séquence est traitée d'un coup.
02

Query, Key, Value

Attention is built on a retrieval metaphor. Think of a library: you have a query (what you are looking for), a set of keys (the labels on each book), and values (the actual content inside). Attention scores how well your query matches each key, then returns a weighted mix of the values.

L'attention repose sur une métaphore de recherche. Imaginez une bibliothèque : vous avez une requête (ce que vous cherchez), un ensemble de clés (les étiquettes sur chaque livre) et des valeurs (le contenu réel). L'attention mesure à quel point votre requête correspond à chaque clé, puis retourne un mélange pondéré des valeurs.

Q
Query
A projection of the current token asking which other tokens are relevant to it right now.
Une projection du token courant qui demande quels autres tokens lui sont pertinents en ce moment.
K
Key
A projection of each token advertising its content. The dot product between Q and K produces the attention score.
Une projection de chaque token qui annonce son contenu. Le produit scalaire entre Q et K produit le score d'attention.
V
Value
The actual information a token contributes to the output. Once the weights are computed from Q and K, the values are summed accordingly.
L'information réelle qu'un token apporte à la sortie. Une fois les poids calculés depuis Q et K, les valeurs sont sommées en conséquence.
Attention(Q, K, V) = softmax( Q · Kᵀ / √dₖ ) · V

Reading left to right: compute dot-product similarity between every Q-K pair, divide by the square root of the key dimension to keep gradients stable, apply softmax to turn the scores into probabilities, then take a weighted sum of the values. Each token ends up with a new representation built from the whole sequence.

En lisant de gauche à droite : calculer la similarité produit scalaire entre chaque paire Q-K, diviser par la racine carrée de la dimension des clés pour stabiliser les gradients, appliquer un softmax pour convertir les scores en probabilités, puis faire la somme pondérée des valeurs. Chaque token obtient une nouvelle représentation construite depuis toute la séquence.

The √dₖ factor is a small but necessary detail. Without it, dot products grow large in high dimensions and push the softmax into near-zero gradient regions, which stalls learning.

Le facteur √dₖ est un détail petit mais nécessaire. Sans lui, les produits scalaires deviennent grands en haute dimension et poussent le softmax dans des zones à gradient quasi nul, ce qui bloque l'apprentissage.

The attention heatmap

La carte de chaleur d'attention

The clearest way to see attention at work is a heatmap. Each row is a token acting as query. Each column is a token acting as key. The intensity of each cell shows the attention weight: how much the query token attends to that key token when building its output representation. The weights shown are illustrative, simplified to highlight key patterns.

La façon la plus claire de voir l'attention en action est une carte de chaleur. Chaque ligne est un token faisant office de requête. Chaque colonne est un token faisant office de clé. L'intensité de chaque cellule montre le poids d'attention. Les valeurs affichées sont illustratives, simplifiées pour mettre en évidence les patterns clés.

04

Multi-head attention

L'attention multi-têtes

A single attention head can only focus on one type of relationship at a time. Real transformers run several heads in parallel, each with its own Q, K, V projection matrices. Each head learns to specialize in a different aspect of language: one tracks syntactic subject-verb agreement, another spots co-references, another is sensitive to local positional patterns.

Une seule tête d'attention ne peut se concentrer que sur un type de relation à la fois. Les vrais transformers font tourner plusieurs têtes en parallèle, chacune avec ses propres matrices de projection Q, K, V. Chaque tête apprend à se spécialiser dans un aspect différent du langage : l'une suit les accords syntaxiques sujet-verbe, une autre repère les coréférences, une autre est sensible aux patterns positionnels locaux.

All heads run in parallel on the same input. Their outputs are concatenated and projected back to the model dimension. The result is a richer representation of each token's context than any single head could provide on its own.

Toutes les têtes tournent en parallèle sur la même entrée. Leurs sorties sont concaténées et projetées vers la dimension du modèle. Le résultat est une représentation plus riche du contexte de chaque token que n'importe quelle tête seule ne pourrait fournir.

05

Where this fits in a transformer

La place de l'attention dans un transformer

Attention is the core operation inside every transformer layer. After attention, each token's representation has been updated to reflect what it learned from attending to the rest of the sequence. A feed-forward network then refines that representation further.

L'attention est l'opération centrale dans chaque couche de transformer. Après l'attention, la représentation de chaque token a été mise à jour pour refléter ce qu'il a appris en regardant le reste de la séquence. Un réseau feed-forward affine ensuite cette représentation.

This whole block (attention followed by feed-forward) is stacked 12, 24, or 96 times. Each layer does not start from scratch: it receives the representations produced by the previous layer and builds on them. Layer 1 might learn basic co-occurrences. Layer 6 might capture syntactic patterns. Layer 24 is already operating on something closer to abstract meaning. The depth is what gives transformers their expressive power.

Ce bloc entier (attention puis feed-forward) est empilé 12, 24 ou 96 fois. Chaque couche ne repart pas de zéro : elle reçoit les représentations produites par la couche précédente et s'appuie dessus. La couche 1 peut apprendre des co-occurrences basiques. La couche 6 peut capturer des patrons syntaxiques. La couche 24 opère déjà sur quelque chose qui ressemble à du sens abstrait. C'est la profondeur qui donne aux transformers leur puissance expressive.

06

What to remember

Ce qu'il faut retenir

Attention in five sentences.

L'attention en cinq phrases.

  1. Attention solves the context problem: every token can look at every other token in a single parallel step, with no distance penalty.
  2. L'attention résout le problème de contexte : chaque token peut regarder tous les autres en une seule étape parallèle, sans pénalité de distance.
  3. Query, Key, Value is a retrieval metaphor: Q asks what it needs, K advertises what each token has, V delivers the content when selected.
  4. Query, Key, Value est une métaphore de recherche : Q demande ce dont il a besoin, K annonce ce que chaque token contient, V livre le contenu quand il est sélectionné.
  5. Attention weights are dot-product similarities, divided by √dₖ for stability, then passed through softmax to form a probability distribution over the sequence.
  6. Les poids d'attention sont des similarités produit scalaire, divisées par √dₖ pour la stabilité, puis passées par un softmax pour former une distribution de probabilité sur la séquence.
  7. Multi-head attention runs several heads in parallel, each specializing in a different linguistic relationship, then merges them.
  8. L'attention multi-têtes fait tourner plusieurs têtes en parallèle, chacune spécialisée dans une relation linguistique différente, puis les fusionne.
  9. Stacking transformer layers builds progressively richer representations: early layers see local patterns, deeper layers operate on abstract meaning.
  10. L'empilement des couches construit des représentations de plus en plus riches : les premières couches voient des patterns locaux, les couches profondes opèrent sur du sens abstrait.
