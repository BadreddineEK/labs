01

Why RNNs weren't enough

Before transformers, language models processed text sequentially — one word at a time, left to right. Recurrent networks (RNNs, LSTMs) maintained a hidden state that accumulated context as they moved through the sequence. The problem: by the time the model reached word 50, the signal from word 1 had been diluted through 49 multiplicative steps.

Long-range dependencies — like the relationship between a pronoun and its antecedent five sentences earlier — were nearly impossible to preserve. The model's memory bottleneck was the hidden state vector, a single fixed-size representation of all prior context.

⏩
Sequential processing
RNNs read one token at a time. Processing can't be parallelized across a sequence, making training on long texts extremely slow.
📉
Vanishing gradients
Gradients shrink exponentially as they travel back through long sequences. Early tokens receive almost no useful learning signal.
🔗
The attention fix
Attention lets every token directly attend to every other token in a single step — no bottleneck, no sequential dependency, full parallelism.
02

Query, Key, Value

Attention is built on a simple retrieval metaphor. Think of a library: you have a query (what you're looking for), a set of keys (the labels on each book), and values (the actual content inside). Attention scores how well your query matches each key, then returns a weighted mix of the values.

Q
Query
"What am I looking for?" — A projection of the current token asking which other tokens are relevant to it.
K
Key
"What do I contain?" — A projection of each token advertising its content to incoming queries.
V
Value
"What do I give if selected?" — The actual information a token contributes to the output if its key matched the query.
Attention(Q, K, V) = softmax( Q · Kᵀ / √dₖ ) · V
dot-product similarity → scaled → softmax weights → weighted sum of values

The √dₖ scaling factor prevents the dot products from growing too large in high dimensions, which would push the softmax into regions with near-zero gradients. It's a small but essential stabilization trick.

04

Multi-head attention

A single attention head can only focus on one type of relationship at a time. Transformers run multiple heads in parallel — each with its own Q, K, V projections. Each head learns to specialize in a different aspect of language: one might track syntactic dependencies, another co-references, another positional patterns.

The outputs of all heads are concatenated and projected back to the model dimension. The result is a rich, multi-perspective representation of each token's context — far more expressive than any single attention pattern.

05

Where this fits

Attention is the core operation inside every transformer layer. Stacked 12, 24, or 96 times deep, each layer refines the representations produced by the one before — progressively building richer contextual embeddings until the final layer produces output that reflects the full meaning of the sequence.

06

What to remember

Attention in five sentences.

  1. Attention lets every token look at every other token in a single parallel operation — no sequential bottleneck.
  2. Query, Key, Value is a retrieval metaphor: Q asks, K advertises, V delivers.
  3. Attention weights are dot-product similarities, scaled and softmaxed into a probability distribution over the sequence.
  4. Multi-head attention runs several heads in parallel, each specializing in a different type of linguistic relationship.
  5. Stacked transformer layers build progressively richer contextual representations — the foundation of every modern LLM.
