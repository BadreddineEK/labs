LLM Agent Anatomy

A bare LLM answers once and stops. An agent reasons, picks tools, observes results, and loops — until the task is done. Here is exactly what happens inside that loop.

6Sections
~6 minRead

01

LLM vs Agent — what changes

A plain LLM is a stateless function: you send a prompt, it returns text. That's it. It cannot browse the web, run code, query a database, or remember what happened ten minutes ago.

An agent wraps the same LLM inside a control loop. The model is no longer the product — it becomes the reasoning engine that decides what to do next. The agent supplies it with tools, memory, and a feedback mechanism so it can iterate until the task is solved.

Bare LLM
One-shot: prompt → text
No tools, no actions
No memory between calls
Cannot verify its own output
Hallucinate freely
LLM Agent
Loop: reason → act → observe
Tools: web, code, APIs, DB…
Short-term + long-term memory
Observes results and self-corrects
Grounded in real-world feedback

Reason

The LLM looks at the task, the conversation history, and the available tools. It writes a Thought — an internal monologue that makes its reasoning explicit. This is the most important step: the quality of the thought directly determines the quality of the action that follows.

Example thought Thought: The user wants the current EUR/USD rate. My training data is outdated. I should use the search tool to get a real-time quote rather than guessing.
03

Tools — the agent's hands

Without tools, an agent is still just a LLM. Tools are the bridge between the model's reasoning and the real world. Each tool is a function signature the LLM can call by name — it never executes code directly, it only returns structured JSON describing which tool to invoke and with what parameters.

🔍
Web Search
Query a search engine and retrieve live results. Defeats knowledge cutoff.
🐍
Code Executor
Run Python snippets in a sandbox. Essential for math, data processing, plots.
📄
RAG / Retriever
Query a private knowledge base. Grounds answers in your own documents.
🗄️
SQL / Database
Translate natural language to SQL and fetch structured data in real time.
🌐
HTTP / APIs
Call any REST endpoint — weather, finance, CRMs, internal microservices.
📅
Calendar / Email
Read and write real-world systems. Enables fully autonomous task delegation.

The LLM sees each tool as a JSON schema: name, description, and typed parameters. It outputs a structured tool_call object — the execution layer (LangChain, LlamaIndex, etc.) runs the actual function and feeds the result back as an Observation.

04

Memory — what the agent remembers

Memory is what separates an agent from a stateless chatbot. Agents use four types of memory, each with a different scope and latency. Most production agents combine at least two of them.

📝
In-context (working memory)
The current prompt window: task description, conversation history, tool results. Fast but bounded — when the window fills up, old content is dropped.
Volatile
🗂️
External (long-term memory)
A vector database or key-value store that persists across sessions. The agent retrieves relevant memories at the start of each task via semantic search.
Persistent
🎞️
Episodic (trace memory)
A log of past Thought → Action → Observation traces. Used for self-reflection: the agent can learn from its own failures in previous runs.
Semi-persistent
⚙️
Procedural (skills / tools)
The tool schemas and system prompt encode what the agent knows how to do — implicit procedural memory that shapes every decision.
Static
05

A full agent trace

Here is what a complete ReAct trace looks like for the question: "What is the current EUR/USD rate and how does it compare to last week?"

💭
Thought #1
I need the current EUR/USD exchange rate. My knowledge cutoff makes this unreliable from memory. I'll call the search tool.
⚡
Action #1
search({ query: "EUR USD exchange rate today April 2026" })
👁️
Observation #1
EUR/USD = 1.0842 as of April 7, 2026 09:15 UTC. Source: Reuters.
💭
Thought #2
Good. Now I need last week's rate (March 31) to compute the delta. I'll run a second search.
⚡
Action #2
search({ query: "EUR USD exchange rate March 31 2026" })
👁️
Observation #2
EUR/USD closed at 1.0791 on March 31, 2026. Source: Bloomberg.
💭
Thought #3
Delta = 1.0842 − 1.0791 = +0.0051 (+0.47%). I have enough to answer. No more tool calls needed.
✅
Final Answer
EUR/USD is currently 1.0842 (Apr 7). Last week it was 1.0791 (Mar 31). The euro strengthened by +0.47% over the past week.
06

What to remember

🔁
The loop is the product
The LLM is not the agent — the loop is. The model is just the reasoning engine inside it.
🛑
Stopping condition matters
Without a clear stopping criterion an agent loops indefinitely. Always define a max-step budget.
🧰
Tools define capability
Adding a code executor multiplies what an agent can do by an order of magnitude.
🔍
Observability is critical
Always log the full Thought → Action → Observation trace. It's the only way to debug a wrong answer.
⚠️
Failure modes are different
Agents fail in novel ways: infinite loops, wrong tool selection, context overflow. Classical evals don't cover this.
📐
Prompt engineering still matters
The system prompt defines tools, stopping rules, and output format. A weak system prompt produces a weak agent.

The agent in five sentences.

  1. The LLM is the reasoning core — it writes thoughts and decides actions, but never executes them directly.
  2. Tools extend its reach — web search, code execution, databases. Each is a typed function the model calls by name.
  3. Observations close the loop — tool results are fed back as context so the model can update its plan and iterate.
  4. Memory gives it persistence — in-context for short tasks, external vector stores for long-running agents.
  5. The trace is everything — Thought → Action → Observation is what you debug, eval, and optimize in production.
