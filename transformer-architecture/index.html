The 6 building blocks

Les 6 blocs fondamentaux

What the Transformer is made of

De quoi est fait le Transformer

Six operations stacked and repeated. Understand each one and the whole becomes clear.

Six opérations empilées et répétées. Comprendre chacune suffit à saisir le tout.

01

TokenizationTokenisation

Text is split into sub-word tokens. Each token gets a unique ID. "unbelievable" becomes ["un","believ","able"]. The model never sees raw text.

Le texte est découpé en sous-mots. Chaque token reçoit un identifiant unique. "incroyable" devient ["in","croy","able"]. Le modèle ne voit jamais le texte brut.

"Hello world" → [15496, 995]

02

Positional EncodingEncodage positionnel

Attention has no sense of order, so position is injected via sinusoidal waves: unique vectors added to each token embedding that encode its place in the sequence.

L'attention n'a pas de sens de l'ordre, la position est donc injectée via des ondes sinusoïdales: des vecteurs uniques ajoutés à chaque embedding qui encodent sa place dans la séquence.

PE(pos,2i) = sin(pos / 10000^(2i/d))

03

Self-Attention

Each token asks: which other tokens should I pay attention to? It computes scores by comparing Query and Key vectors, then uses those scores to weight the Values.

Chaque token demande: quels autres tokens méritent mon attention? Il calcule des scores en comparant les vecteurs Query et Key, puis utilise ces scores pour pondérer les Values.

Attention(Q,K,V) = softmax(QKᵀ/√d)V

04

Multi-Head Attention

Self-attention runs h times in parallel with different learned projections. Each head can focus on different linguistic relationships at the same time.

L'auto-attention s'exécute h fois en parallèle avec des projections apprises distinctes. Chaque tête peut se concentrer sur différentes relations linguistiques en même temps.

MultiHead = Concat(head₁,...,headₜ)Wᴴ

05

Layer Normalization

After each sub-layer, activations are normalized to zero mean and unit variance. This stabilizes training and allows much deeper stacking without collapse.

Après chaque sous-couche, les activations sont normalisées à moyenne nulle et variance unitaire. Cela stabilise l'entraînement et permet un empilement bien plus profond.

LayerNorm(x) = (x - μ) / (σ + ε) · γ + β

06

Feed-Forward Network

A position-wise MLP applied identically to every token. It expands to 4x the model dimension, applies GELU, then projects back. Most parameters live here.

Un MLP position par position appliqué identiquement à chaque token. Il s'étend à 4x la dimension du modèle, applique GELU, puis reprojette. C'est là que résident la plupart des paramètres.

FFN(x) = max(0, xW₁+b₁)W₂+b₂

Self-Attention deep dive

Plongée dans le self-attention

Query, Key, Value

Query, Key, Value

Think of it as a soft database lookup. Q is what a token is searching for, K is what each token offers, and V is what actually gets retrieved when a match is found.

Pensez-y comme une recherche souple dans une base de données. Q est ce qu'un token cherche, K est ce que chaque token propose, et V est ce qui est réellement récupéré quand une correspondance est trouvée.

Q

Query

What this token is looking for. Like a search query in a database.

Ce que ce token cherche. Comme une requête de recherche dans une base de données.

K

Key

What each token offers. The index that Q is matched against.

Ce que chaque token propose. L'index sur lequel Q est comparé.

V

Value

The actual content retrieved when the Q·K match is high.

Le contenu réellement récupéré quand la correspondance Q·K est élevée.

Step by step: "Le chat mange la souris" Etape par etape: "Le chat mange la souris"

1

Linear projections. Each token embedding is multiplied by 3 weight matrices (Wq, Wk, Wv) to produce its Q, K, and V vectors.

Projections linéaires. Chaque embedding de token est multiplié par 3 matrices de poids (Wq, Wk, Wv) pour produire ses vecteurs Q, K et V.

2

Dot-product scores. Q of token i multiplied by K of token j gives a raw score indicating how much i should attend to j.

Scores produit scalaire. Q du token i multiplié par K du token j donne un score brut indiquant à quel point i doit porter attention à j.

3

Scale and softmax. Dividing by √d_k prevents the dot products from growing too large; softmax converts scores into weights that sum to 1.

Mise à l'échelle et softmax. Diviser par √d_k empêche les produits scalaires de devenir trop grands; le softmax convertit les scores en poids dont la somme vaut 1.

4

Weighted sum of V. Each V is multiplied by its attention weight. The sum is the new representation of token i, enriched with context from the full sequence.

Somme pondérée des V. Chaque V est multiplié par son poids d'attention. La somme est la nouvelle représentation du token i, enrichie du contexte de la séquence entière.

Full architecture

Architecture complète

Encoder & Decoder

Encodeur et Décodeur

The original Transformer has two stacks. Modern LLMs like GPT use only the decoder. BERT uses only the encoder. Encoder-decoder models are used for translation and summarization.

Le Transformer original a deux piles. Les LLMs modernes comme GPT n'utilisent que le décodeur. BERT n'utilise que l'encodeur. Les modèles encodeur-décodeur sont utilisés pour la traduction et le résumé.

Key equations

Les équations clés

The math behind it

Les maths derrière

Four equations define the entire architecture.

Quatre équations définissent toute l'architecture.

Scaled Dot-Product Attention

Attention produit scalaire mis à l'échelle

Attention(Q,K,V) = softmax( QKᵀ / √d_k ) · V

The core operation. Dividing by √d_k prevents the dot products from growing too large, which would push softmax into regions with very small gradients.

L'opération centrale. Diviser par √d_k empêche les produits scalaires de devenir trop grands, ce qui pousserait le softmax dans des zones à gradients très faibles.

Multi-Head Attention

MultiHead(Q,K,V) = Concat(head₁,...,headₜ) · Wᴴ
headᵢ = Attention(QWᵢQ, KWᵢK, VWᵢV)

Each head learns its own Q/K/V projections, allowing the model to attend to different subspaces at the same time.

Chaque tête apprend ses propres projections Q/K/V, ce qui permet au modèle de porter attention à différents sous-espaces simultanément.

Position-wise FFN

FFN position par position

FFN(x) = max(0, xW₁ + b₁) · W₂ + b₂

Applied independently to each position. The hidden layer is typically 4x wider than the model dimension. This is where most parameters live.

Appliqué indépendamment à chaque position. La couche cachée est typiquement 4x plus large que la dimension du modèle. C'est là que résident la plupart des paramètres.

Positional Encoding

Encodage positionnel

PE(pos,2i) = sin(pos / 10000^(2i/d_model))
PE(pos,2i+1) = cos(pos / 10000^(2i/d_model))

Sine and cosine waves at different frequencies encode position. Modern models replace this with RoPE or ALiBi for better length generalization.

Des ondes sinus et cosinus à différentes fréquences encodent la position. Les modèles modernes remplacent cela par RoPE ou ALiBi pour une meilleure généralisation sur la longueur.

Timeline

How one architecture took over all of AI in eight years.

Comment une seule architecture a pris le contrôle de tout l'IA en huit ans.

2017

Attention Is All You Need

Vaswani et al. at Google Brain publish the original paper. Eight authors, one architecture.

Vaswani et al. chez Google Brain publient l'article original. Huit auteurs, une architecture.

2018

BERT & GPT-1

Google's BERT (encoder-only, bidirectional) and OpenAI's GPT (decoder-only, autoregressive) establish the two main paths forward.

Le BERT de Google (encodeur seul, bidirectionnel) et le GPT d'OpenAI (décodeur seul, autorégressif) établissent les deux grandes directions à suivre.

2020

GPT-3 (175B parameters)

Few-shot learning emerges. Same architecture, scaled up 1000x. The broader world takes notice.

L'apprentissage few-shot émerge. Même architecture, multipliée par 1000. Le monde entier commence à s'y intéresser.

2022

ChatGPT & the RLHF era

Instruction fine-tuning combined with RLHF turns a raw language model into an assistant. One million users in five days.

Le fine-tuning par instruction combiné au RLHF transforme un modèle brut en assistant. Un million d'utilisateurs en cinq jours.

2024-26

MoE, long context, reasoning

GPT-4, Claude 3, Gemini Ultra, Llama 3: all Transformers. Context windows reach 1M+ tokens. The architecture holds.

GPT-4, Claude 3, Gemini Ultra, Llama 3: tous des Transformers. Les fenêtres de contexte atteignent 1M+ tokens. L'architecture tient bon.

Where this matters

Où ca compte vraiment

What the Transformer powers today

Ce que le Transformer alimente aujourd'hui

Knowing the architecture helps you make better decisions when deploying, fine-tuning, or building on top of these models.

Connaître l'architecture aide à prendre de meilleures décisions lors du déploiement, du fine-tuning ou de la construction par-dessus ces modèles.

Use case 1

Inference cost

Coût d'inférence

Self-attention scales quadratically with sequence length. A 4x longer prompt costs roughly 16x more to process. Understanding this lets you design prompts and context windows more deliberately.

L'auto-attention s'adapte quadratiquement à la longueur de la séquence. Un prompt 4x plus long coûte environ 16x plus cher à traiter. Comprendre cela permet de concevoir les prompts de façon plus réfléchie.

Use case 2

Fine-tuning strategies

Stratégies de fine-tuning

LoRA works by adding low-rank adapters to the attention weight matrices (Wq, Wk, Wv, Wo). Knowing the architecture shows you exactly what gets modified and why it is parameter-efficient.

LoRA fonctionne en ajoutant des adaptateurs de bas rang aux matrices de poids d'attention (Wq, Wk, Wv, Wo). Connaître l'architecture montre exactement ce qui est modifié et pourquoi c'est efficace en paramètres.

Use case 3

RAG and retrieval

RAG et retrieval

In RAG pipelines, retrieved chunks are prepended to the decoder's context. The Transformer processes them with the same attention mechanism, so retrieved context competes with the original prompt token by token.

Dans les pipelines RAG, les chunks récupérés sont ajoutés au contexte du décodeur. Le Transformer les traite avec le même mécanisme d'attention, donc le contexte récupéré entre en compétition avec le prompt d'origine token par token.

Summary

Résumé

Key takeaways

Points clés à retenir

🧩

Six operations repeated N times is all a Transformer is: tokenization, positional encoding, self-attention, multi-head projection, layer norm, and feed-forward.

Six opérations répétées N fois, c'est tout ce qu'est un Transformer: tokenisation, encodage positionnel, auto-attention, projection multi-tête, norme de couche et feed-forward.

🔍

Self-attention lets every token attend directly to every other token in a single parallel step. This is what replaced the sequential bottleneck of RNNs.

L'auto-attention permet à chaque token de porter attention directement à chaque autre en une seule étape parallèle. C'est ce qui a remplacé le goulot d'étranglement séquentiel des RNN.

🧠

Multiple attention heads run in parallel, each learning to focus on different aspects: syntax, co-reference, local structure, and so on.

Plusieurs têtes d'attention s'exécutent en parallèle, chacune apprenant à se concentrer sur des aspects différents: syntaxe, co-référence, structure locale, etc.

📐

Attention complexity is O(n²) in sequence length. Longer contexts cost more to process, which is why efficient attention variants (FlashAttention, sliding window) matter at scale.

La complexité de l'attention est O(n²) en longueur de séquence. Les contextes plus longs coûtent plus cher à traiter, c'est pourquoi les variantes d'attention efficaces (FlashAttention, fenêtre glissante) comptent à grande échelle.

🏗️

GPT-style models are decoder-only. BERT is encoder-only. The full encoder-decoder is used for seq2seq tasks like translation. The choice of stack determines what the model can do.

Les modèles de style GPT sont décodeur uniquement. BERT est encodeur uniquement. L'encodeur-décodeur complet est utilisé pour les tâches seq2seq comme la traduction. Le choix de la pile détermine ce que le modèle peut faire.

📅

The architecture from the 2017 paper is still the foundation of every major model in 2026. The improvements since then are mostly in training recipes, data, and scale.

L'architecture du papier de 2017 est toujours la base de chaque grand modèle en 2026. Les améliorations depuis lors concernent surtout les recettes d'entraînement, les données et l'échelle.
