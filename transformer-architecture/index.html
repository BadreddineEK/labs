The Transformer Architecture

Every modern LLM — GPT, BERT, Llama, Gemini — is built on one paper from 2017: "Attention Is All You Need". Here is exactly what that architecture does, block by block.

6Sections
~7 minRead

01

Why the Transformer replaced RNNs

Before 2017, sequence models were dominated by RNNs and LSTMs. They processed tokens one by one, left to right, passing a hidden state from step to step. This worked — but it had two fundamental limits: they could not be parallelized (each token depends on the previous), and they struggled to maintain context over long sequences because information had to travel through every step to reach the end.

The Transformer fixed both at once. Instead of processing tokens sequentially, it processes the entire sequence in parallel, and instead of passing state through time, it uses attention to directly connect any token to any other token regardless of distance. Same compute, far better results.

RNN / LSTM
Sequential — cannot parallelize
Context fades over long sequences
O(n) steps to connect distant tokens
Slow to train on modern GPUs
Gradient vanishing on deep stacks
Transformer
Fully parallel — all tokens at once
Every token attends to every other
O(1) steps between any two tokens
Scales linearly with GPU count
Residual connections prevent vanishing

Input Embeddings + Positional Encoding

Each token (word or sub-word) is first converted to a dense vector of dimension d_model (typically 512 or 768) by an embedding lookup table. But unlike RNNs, the Transformer has no inherent sense of order — so a positional encoding is added to each embedding. The original paper uses sine and cosine functions of different frequencies, giving each position a unique signature the model can learn from.

PE(pos, 2i) = sin(pos / 10000^(2i/d_model)) PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
03

Self-Attention — click a token

Self-attention lets each token ask a question (Query) about every other token (Key), and retrieve information from them (Value) weighted by relevance. The result: a new representation of each token that is enriched with context from the entire sequence. Click a token below to see which other tokens it attends to most.

Select a token to see its attention pattern:
"The" mostly attends to nearby tokens. Function words spread attention broadly — they depend on surrounding context to carry meaning.

The attention score between query q and key k is computed as softmax(q·kᵀ / √d_k). The √d_k scaling prevents scores from becoming too large in high-dimensional spaces, which would push softmax into regions with near-zero gradients.

04

Multi-Head Attention — why multiple heads?

Running a single attention pass would force the model to compress all relationships into one representation. Multi-Head Attention runs several attention operations in parallel — each head projects Q, K, V into a different lower-dimensional subspace, learns a different type of relationship, then all heads are concatenated and projected back.

H1
Syntactic
Tracks subject–verb agreement and grammatical dependencies between words.
H2
Coreference
Links pronouns to their antecedents — connects "it" back to the noun it refers to.
H3
Positional
Attends strongly to adjacent tokens — learns local sequential patterns.
H4
Semantic
Captures word meaning similarity — connects words with related topics.
H5
Long-range
Bridges distant tokens — maintains topic coherence across paragraphs.
H6–N
Emergent
Learned patterns that don't map cleanly to human-interpretable categories.
🔗
Concatenate → Linear projection
All head outputs are concatenated into a single vector, then multiplied by a learned weight matrix W_O to project back to d_model. This final step lets the model decide how to blend the information each head found.
05

Encoder · Decoder · Encoder-Decoder

The original "Attention Is All You Need" paper introduced an encoder-decoder architecture for translation. But modern models use only part of it — and which part they use defines what they are good at.

Encoder-only models

The encoder reads the entire input sequence at once — each token attends to all others (bidirectional). This produces rich contextual representations ideal for understanding tasks. No text generation, but excellent classification, NER, and embedding quality.

Decoder-only models

The decoder uses causal (masked) self-attention — each token can only attend to previous tokens, not future ones. This auto-regressive design is perfect for text generation: predict the next token, append it, repeat. Every GPT-family model is decoder-only.

Encoder-Decoder models

The original Transformer. The encoder reads the full source (e.g. French sentence), the decoder generates the target (e.g. English) token by token, attending to both its own previous output and the encoder's representation via cross-attention. Ideal for seq2seq tasks: translation, summarization, code generation from specs.

06

What to remember

⚡
Parallelism is the superpower
Processing all tokens at once instead of step by step is what made scaling Transformers to billions of parameters feasible on modern GPUs.
🔗
Residuals prevent degradation
Every sub-layer adds its output to its input (x + sublayer(x)). This lets gradients flow directly through dozens of layers without vanishing.
📐
d_model is the backbone
The embedding dimension d_model (512, 768, 1024…) runs through every layer. It's the single number that most controls model capacity and compute.
🧮
Attention is O(n²)
Every token attends to every other — so compute scales quadratically with sequence length. This is why context windows are expensive, and why Flash Attention matters.
🎯
The FFN is the memory
Research shows the Feed-Forward layers store factual knowledge. Attention routes information; FFN holds it. This is why larger FFN layers improve knowledge-intensive tasks.
🔄
Same block, stacked deep
GPT-4 is estimated at ~96 layers of the same block. No architectural magic — just scale, data, and compute applied to the same elegant structure.

The Transformer in five sentences.

  1. Parallelism over sequence — all tokens are processed simultaneously, not one by one, which is why Transformers scale so well on hardware.
  2. Attention is the routing mechanism — it lets every token directly query information from any other token in O(1) steps.
  3. Multiple heads see multiple relationships — syntax, coreference, semantics, position — each head learns a different aspect of language.
  4. Residuals and LayerNorm make depth possible — without them, gradients would vanish through dozens of stacked blocks.
  5. Encoder reads, Decoder generates — choose encoder-only for understanding, decoder-only for generation, encoder-decoder for translation and summarization.
