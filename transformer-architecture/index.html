The 6 building blocks

What the Transformer is made of

The architecture is deceptively simple — 6 operations stacked and repeated. Understand each one and the whole becomes clear.

01

Tokenization

Text is split into sub-word tokens. Each token gets a unique ID. "unbelievable" → ["un","believ","able"]. The model never sees raw text.

"Hello world" → [15496, 995]

02

Positional Encoding

Attention has no sense of order. So we inject position via sinusoidal waves — unique vectors added to each token embedding that encode its place in the sequence.

PE(pos,2i) = sin(pos / 10000^(2i/d))

03

Self-Attention

Each token asks: "which other tokens should I pay attention to?" It computes scores by comparing Query and Key vectors, then uses those scores to weight the Values.

Attention(Q,K,V) = softmax(QKᵀ/√d)V

04

Multi-Head Attention

Run self-attention h times in parallel with different learned projections. Each "head" can focus on different linguistic relationships simultaneously.

MultiHead = Concat(head₁,...,headₕ)Wᴼ

05

Layer Normalization

After each sub-layer, normalize activations to zero mean and unit variance. This stabilizes training and allows much deeper stacking.

LayerNorm(x) = (x - μ) / (σ + ε) · γ + β

06

Feed-Forward Network

A position-wise MLP applied identically to every token. It expands to 4× the model dimension, applies ReLU/GELU, then projects back. This is where most parameters live.

FFN(x) = max(0, xW₁+b₁)W₂+b₂

Self-Attention deep dive

Query, Key, Value — explained

The hardest part to intuit. Think of it as a soft database lookup: Q is what you're searching for, K is what each token offers, V is what you actually retrieve.

Q

Query

What this token is looking for. Like a search query in a database.

K

Key

What each token offers. The "index" that Q is matched against.

V

Value

The actual content retrieved if the Q·K match is high.

Step by step — "Le chat mange la souris"

1

Linear projections — Each token embedding is multiplied by 3 weight matrices (Wq, Wk, Wv) to get its Q, K and V vectors.

2

Dot-product scores — Q of token i × K of token j → a raw score indicating how much i should attend to j.

3

Scale + Softmax — Divide by √d_k to prevent vanishing gradients, then softmax to get weights that sum to 1.

4

Weighted sum of V — Multiply each V by its attention weight. Sum = the new representation of token i, enriched with context.

Full architecture

Encoder & Decoder

The original Transformer has two stacks. Modern LLMs like GPT use only the decoder. BERT uses only the encoder. Encoder-decoder is used for translation and summarization.

Key equations

The math behind it

Four equations define the entire architecture. Understand these and you understand every LLM built since 2017.

Scaled Dot-Product Attention

Attention(Q,K,V) = softmax( QKᵀ / √d_k ) · V

The core operation. Divide by √d_k to prevent the dot products from growing too large, which would push softmax into regions with tiny gradients.

Multi-Head Attention

MultiHead(Q,K,V) = Concat(head₁,...,headₕ) · Wᴼ
headᵢ = Attention(QWᵢQ, KWᵢK, VWᵢV)

Each head learns its own Q/K/V projections, allowing the model to attend to different subspaces simultaneously.

Position-wise FFN

FFN(x) = max(0, xW₁ + b₁) · W₂ + b₂

Applied independently to each position. The hidden layer is typically 4× wider than the model dimension — this is where most parameters live.

Positional Encoding

PE(pos,2i) = sin(pos / 10000^(2i/d_model))
PE(pos,2i+1) = cos(pos / 10000^(2i/d_model))

Sine and cosine waves at different frequencies encode position. Modern models replace this with RoPE or ALiBi for better length generalization.

From paper to paradigm

Timeline

How a single architecture took over all of AI in eight years.

2017

Attention Is All You Need

Vaswani et al. at Google Brain publish the original paper. 8 authors, one architecture that would change everything.

2018

BERT & GPT-1

Google's BERT (encoder-only, bidirectional) and OpenAI's GPT (decoder-only, autoregressive) show the two paths forward.

2020

GPT-3 (175B parameters)

Few-shot learning emerges. The same architecture, scaled × 1000. The world notices.

2022

ChatGPT & the RLHF era

Instruction fine-tuning + RLHF turns a raw language model into an assistant. 1 million users in 5 days.

2024–26

MoE, long context, reasoning

GPT-4, Claude 3, Gemini Ultra, Llama 3 — all Transformers. Context windows reach 1M+ tokens. The architecture holds.
